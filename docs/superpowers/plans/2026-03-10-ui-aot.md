# ISynergy.Framework.UI AOT Compatibility Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `ISynergy.Framework.UI` (the shared base library) AOT-compatible by replacing its two core reflection-based patterns — assembly-scanning registration and `AppDomain`-based view-type resolution — with a source-generator-driven compile-time registration table.

**Architecture:** A new `ISynergy.Framework.UI.SourceGenerator` Roslyn incremental generator (targeting `netstandard2.0`) is bundled as an analyzer inside the existing UI NuGet. It scans the consuming assembly's compilation for `IView`, `IWindow`, and `IViewModel` implementors and emits a single `RegisterUITypes(IServiceCollection)` extension method that replaces all calls to `RegisterAssemblies` / `ToViewTypes` / `ToWindowTypes` / `ToViewModelTypes`. The runtime library's `ViewModelExtensions.GetRelatedViewType` is replaced with a generated compile-time `Dictionary<string, Type>` lookup; `ReflectionExtensions` methods are annotated with `[RequiresUnreferencedCode]` as intermediate guardrails.

**Tech Stack:** .NET 10 / C# 14, Roslyn Incremental Source Generators (`IIncrementalGenerator`), `Microsoft.CodeAnalysis.CSharp` 4.x, MSTest for generator tests, central package management via `Directory.Packages.props`.

## Implementation Steps

- [ ] **Step 1 — Create the source generator project**
  - Create `src/ISynergy.Framework.UI.SourceGenerator/ISynergy.Framework.UI.SourceGenerator.csproj` targeting `netstandard2.0`
  - Add `Microsoft.CodeAnalysis.CSharp` dependency via `Directory.Packages.props`
  - Configure `<IsRoslynComponent>true</IsRoslynComponent>` and `<EnforceExtendedAnalyzerRules>true</EnforceExtendedAnalyzerRules>`

- [ ] **Step 2 — Implement `UIRegistrationGenerator`**
  - Use `IIncrementalGenerator.Initialize` with `SyntaxValueProvider.CreateSyntaxProvider` to find all non-abstract classes implementing `IView`, `IWindow`, or `IViewModel`
  - Collect `(ConcreteType, InterfaceType)` pairs in an `IncrementalValuesProvider`
  - Emit a `UIGeneratedRegistrations` partial static class with method:
    ```csharp
    public static IServiceCollection AddUITypes(this IServiceCollection services)
    ```
    containing `services.TryAddTransient<IMyView, MyView>(); services.TryAddTransient<MyView>();` lines for every discovered pair.

- [ ] **Step 3 — Implement `ViewModelViewMapGenerator`**
  - Collect all `IViewModel` implementors and their `GetRelatedView()` string (computed at generator time by name manipulation mirroring `ViewModelExtensions.GetRelatedView`)
  - Emit a `ViewModelViewMap` static class with:
    ```csharp
    public static readonly IReadOnlyDictionary<string, Type> Map = new Dictionary<string, Type> { ... };
    ```
  - Emit `GetRelatedViewType(this string name)` extension method (in consuming assembly namespace) that uses the map

- [ ] **Step 4 — Add `[RequiresUnreferencedCode]` annotations to `ReflectionExtensions`**
  - Annotate every public method in `ReflectionExtensions` that calls `GetExportedTypes` or `AppDomain.CurrentDomain`
  - Add `[RequiresDynamicCode("...")]` where `MakeGenericType` or similar is transitively called

- [ ] **Step 5 — Add `[RequiresUnreferencedCode]` annotations to `ViewModelExtensions`**
  - Annotate `GetRelatedViewType(string)`, `CancelAllCommands`, `ResetAllCommandStates`
  - Add new AOT-safe overloads that accept the generated map / do not use reflection for property scanning

- [ ] **Step 6 — Annotate `SingletonHelper<T>`**
  - Add `where T : new()` constraint already exists; add `[DynamicallyAccessedMembers(DynamicallyAccessedMemberTypes.PublicParameterlessConstructor)]` attribute

- [ ] **Step 7 — Wire generator into `ISynergy.Framework.UI.csproj`**
  - Add `<ProjectReference Include="..\ISynergy.Framework.UI.SourceGenerator\..." OutputItemType="Analyzer" ReferenceOutputAssembly="false" />`
  - Add AOT-safe overload to `ServiceCollectionExtensions.RegisterAssemblies` that delegates to `UIGeneratedRegistrations.AddUITypes`

- [ ] **Step 8 — Add new AOT-safe `RegisterAssemblies` overload to `ServiceCollectionExtensions`**
  - Signature: `RegisterAssemblies(this IServiceCollection services)` — calls `AddUITypes(services)` from generated code
  - Mark old `RegisterAssemblies(Assembly, Func<AssemblyName,bool>)` overload with `[RequiresUnreferencedCode]`

- [ ] **Step 9 — Write generator unit tests**
  - Use `Microsoft.CodeAnalysis.CSharp.SourceGenerators.Testing` in test project
  - Test: empty input → empty registrations
  - Test: one `IView` implementor → correct registration emitted
  - Test: one `IViewModel` implementor → correct view-map entry emitted
  - Test: generic ViewModel → correct name stripping

- [ ] **Step 10 — Update `Directory.Packages.props`**
  - Add version pin for `Microsoft.CodeAnalysis.CSharp` (minimum 4.x, matching Roslyn shipped with .NET 10 SDK)

## Notes on Platform Scope

- This base library (`ISynergy.Framework.UI`) is consumed by all platform libraries (WinUI, WPF, MAUI, UWP, Blazor).
- The source generator runs in the **consuming application's** build, not in the library build, so it will correctly enumerate the application-level Views and ViewModels.
- The `GetRelatedViewType` call chain used by every platform's `NavigationExtensions.CreatePage<TViewModel>` is the single most critical AOT issue shared across all platforms; fixing it here fixes all downstream callers.
- `ViewModelExtensions.CancelAllCommands` and `ResetAllCommandStates` use `GetProperties` reflection; the AOT-safe alternative is to require ViewModels to implement an `ICancellableViewModel` interface that explicitly exposes `CancelAllCommands()` — this is an application-level refactor deferred to a separate plan. For now, `[RequiresUnreferencedCode]` annotation is sufficient.

## Migration Guide

The assembly-scanning `RegisterAssemblies(Assembly, ...)` and `GetRelatedViewType(string)` calls are replaced by a Roslyn incremental source generator (`ISynergy.Framework.UI.SourceGenerator`) that emits `AddUITypes()` and a compile-time view-name dictionary at build time.

### Before

```csharp
// App startup — reflection-based assembly scan
services.RegisterAssemblies(
    Assembly.GetExecutingAssembly(),
    name => name.Name?.StartsWith("MyApp") == true);

// Navigation — runtime assembly walk
var viewType = viewModelName.GetRelatedViewType();
```

### After

```csharp
// App startup — source-generator-emitted registration (no reflection)
services.AddUITypes(); // generated by ISynergy.Framework.UI.SourceGenerator

// Navigation — uses generated compile-time dictionary (fast path)
var viewType = ViewModelViewMap.Map.TryGetValue(viewModelName, out var t)
    ? t
    : viewModelName.GetRelatedViewType(); // [RequiresUnreferencedCode] fallback
```

### Steps to Migrate

1. Remove the `RegisterAssemblies(Assembly, ...)` call from your application startup.
2. Build the project — the bundled `ISynergy.Framework.UI.SourceGenerator` Roslyn analyzer emits `AddUITypes()` in the consuming assembly.
3. Call `services.AddUITypes()` in your DI configuration in place of the old `RegisterAssemblies` call.
4. Update navigation code to use `ViewModelViewMap.Map` (emitted by the generator) for the view-type lookup before falling back to `GetRelatedViewType`.
5. Implement `ICommandProvider` on ViewModels to avoid the `[RequiresUnreferencedCode]`-annotated `CancelAllCommands` / `ResetAllCommandStates` reflection paths.
