# ISynergy.Framework.Mvvm AOT Compatibility Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `ISynergy.Framework.Mvvm` fully AOT-compatible by replacing the `AppDomain.CurrentDomain.GetAssemblies()` + `GetExportedTypes()` assembly-scan in `GetRelatedViewType` with a registration-based lookup, and annotating the remaining runtime-reflection helpers with the appropriate trimmer attributes.

**Architecture:** The assembly-scanning in `ViewModelExtensions.GetRelatedViewType` is the only pattern that requires a structural change — it cannot be fixed by annotation alone because the output (the found `Type`) is used to instantiate views, and AOT cannot trace which types callers may request. The fix replaces the scan with a caller-supplied `IReadOnlyDictionary<string, Type>` registry that consuming applications populate at startup (typically from a source-generated registration). All other issues in the library are annotation-only fixes using `[DynamicallyAccessedMembers]` and `[RequiresUnreferencedCode]`/`[RequiresDynamicCode]`.

**Tech Stack:** .NET 10 / C# 14, `System.Diagnostics.CodeAnalysis` trimmer attributes.

## AOT Issues Found

### Issue 1 — `AppDomain` assembly scan + `GetExportedTypes()` (AOT blocker — structural fix required)

**File:** `Extensions/ViewModelExtensions.cs`
**Lines:** 131–145

```csharp
public static Type? GetRelatedViewType(this string name)
{
    return
        AppDomain.CurrentDomain.GetAssemblies()
            .Where(assemblyFilter)
            .Reverse()
                .Select(assembly => assembly.GetType(name))           // line 137
                .FirstOrDefault(t => t is not null)
            ??
            AppDomain.CurrentDomain.GetAssemblies()
                .Where(assemblyFilter)
                .Reverse()
                .SelectMany(assembly => assembly.GetExportedTypes())  // line 143
                .FirstOrDefault(t => t.Name.Equals(name));
}
```

`AppDomain.CurrentDomain.GetAssemblies()` enumerates all currently-loaded assemblies at runtime. Under AOT the linker trims types that are not reachable from static roots, so the returned list will be incomplete and the `GetExportedTypes()` call will miss trimmed types. Additionally `GetExportedTypes()` itself requires `[RequiresUnreferencedCode]` in .NET AOT. This cannot be fixed with annotations only — there is no `[DynamicallyAccessedMembers]` that covers "all types exported from all assemblies loaded at runtime".

### Issue 2 — `GetProperties()` on runtime `IViewModel` instances (annotation-only fix)

**File:** `Extensions/ViewModelExtensions.cs`
**Lines:** 155–165, 177–187

    var commandProperties = viewModel.GetType()

        .Where(p => typeof(ICancellationAwareCommand).IsAssignableFrom(p.PropertyType));

`viewModel.GetType().GetProperties(...)` walks properties of a runtime-typed object. The concrete ViewModel types are defined in consuming applications and are not known to this library, so these calls require `[RequiresUnreferencedCode]`.

### Issue 3 — `GetCustomAttributes` on enum field (annotation-only fix)

**File:** `ViewModels/ViewModel.cs`
**Lines:** 158–164

```csharp
var fieldInfo = value.GetType().GetField(description);
var attributes = (DisplayAttribute[])fieldInfo.GetCustomAttributes(typeof(DisplayAttribute), false);
```

`value.GetType()` resolves at runtime; `GetField` + `GetCustomAttributes` on the resulting type requires the field's custom attributes to be preserved by the trimmer. The enum type `T` is passed as `Enum` (untyped), so a `[DynamicallyAccessedMembers]` constraint cannot be placed statically — `[RequiresUnreferencedCode]` is the correct fix.

### Issue 4 — `GetInterfaces()` in navigation helpers (annotation-only — low risk)

`Type.GetInterfaces()` is AOT-safe on its own; the interface list is preserved by the trimmer when the type implements those interfaces. No annotation needed here.

## Implementation Plan

### Step 1 — Replace assembly scan in `GetRelatedViewType` with a registry-based lookup

This is the only structural change required. The goal is to preserve the existing public API contract while making it AOT-safe.

- [ ] Add a new internal static class `ViewTypeRegistry` in `Extensions/ViewModelExtensions.cs` (or a dedicated `Registration/ViewTypeRegistry.cs` file):

  ```csharp
  internal static class ViewTypeRegistry
  {
      private static IReadOnlyDictionary<string, Type> _registry =
          new Dictionary<string, Type>(StringComparer.Ordinal);

      internal static void Register(IReadOnlyDictionary<string, Type> registry)
          => _registry = registry;

      internal static Type? TryGet(string name)
          => _registry.TryGetValue(name, out var type) ? type : null;
  }
  ```

- [ ] Add a new public method `RegisterViewTypes(IReadOnlyDictionary<string, Type> viewTypeMap)` in `ViewModelExtensions` that delegates to `ViewTypeRegistry.Register(...)`. This is the method consuming applications call at startup to provide the view-name → type mapping.

  ```csharp
  /// <summary>
  /// Registers the application's view type map for AOT-compatible view resolution.
  /// Call this at application startup before any navigation occurs.
  /// </summary>
  /// <param name="viewTypeMap">
  /// A dictionary mapping view type names (e.g. <c>"UserView"</c>) to their <see cref="Type"/> objects.
  /// </param>
  public static void RegisterViewTypes(IReadOnlyDictionary<string, Type> viewTypeMap)
  {
      ArgumentNullException.ThrowIfNull(viewTypeMap);
      ViewTypeRegistry.Register(viewTypeMap);
  }
  ```

- [ ] Modify `GetRelatedViewType(this string name)` to use the registry as primary lookup, falling back to the assembly scan only when the registry has not been populated (i.e., when running in non-AOT mode):

  ```csharp
  [RequiresUnreferencedCode("Fallback assembly scanning is not AOT-compatible. " +
      "Call ViewModelExtensions.RegisterViewTypes() at startup to enable AOT-safe view resolution.")]
  public static Type? GetRelatedViewType(this string name)
  {
      // Fast AOT-safe path: use the registered map
      var registered = ViewTypeRegistry.TryGet(name);
      if (registered is not null)
          return registered;

      // Slow reflection path (non-AOT only)
      return AppDomain.CurrentDomain.GetAssemblies()
              .Where(assemblyFilter)
              .Reverse()
              .Select(assembly => assembly.GetType(name))
              .FirstOrDefault(t => t is not null)
          ??
          AppDomain.CurrentDomain.GetAssemblies()
              .Where(assemblyFilter)
              .Reverse()
              .SelectMany(assembly => assembly.GetExportedTypes())
              .FirstOrDefault(t => t.Name.Equals(name));
  }
  ```

  The `[RequiresUnreferencedCode]` attribute on the method ensures that any AOT-compiled caller gets a build error unless the registry path is used (callers that populate the registry first will never reach the reflection fallback). If the team prefers a hard break from the reflection fallback in AOT builds, a `#if` guard can eliminate the fallback body entirely under `PublishAot`.

- [ ] Add XML documentation to `GetRelatedViewType` explaining the AOT contract and the `RegisterViewTypes` prerequisite.

### Step 2 — Annotate `CancelAllCommands` and `ResetAllCommandStates` (Issue 2)

- [ ] Add `[RequiresUnreferencedCode("ViewModel command discovery uses runtime reflection over property types.")]` to `CancelAllCommands(this IViewModel viewModel)` in `Extensions/ViewModelExtensions.cs`.
- [ ] Add the same annotation to `ResetAllCommandStates(this IViewModel viewModel)`.
- [ ] Add `using System.Diagnostics.CodeAnalysis;` to `ViewModelExtensions.cs` if not already present.

  **Alternative (preferred for full AOT-safety):** Replace the property-scan with a convention: have `IViewModel` expose an `IEnumerable<ICancellationAwareCommand> GetCancellableCommands()` method that each ViewModel implements explicitly (returning the commands registered in its constructor). The base `ViewModel` class can provide a default empty implementation. This eliminates reflection entirely. This is listed as an optional follow-up and is not required to unblock AOT compilation — the annotation approach is sufficient.

## Source Generator Assessment

No source generator is required for `ISynergy.Framework.Mvvm`. The only assembly-scanning pattern (`GetRelatedViewType`) is replaced by a registry that consuming application startup code populates. Consuming application projects may optionally use the existing CQRS source generator pattern or a similar incremental generator to auto-generate the `RegisterViewTypes(...)` call from their view types — but that generator belongs to the application layer, not to this library.

---

## Migration Guide

The assembly-scanning `GetRelatedViewType` method is replaced with a registry-based lookup. Calling applications must register their view type map at startup before any navigation occurs. `CancelAllCommands`, `ResetAllCommandStates`, and `GetEnumDescription` are annotated with `[RequiresUnreferencedCode]`.

### Before

```csharp
// Navigation relied on assembly scanning at runtime
var viewType = viewModelName.GetRelatedViewType();
// No startup registration required
```

### After

```csharp
// Application startup (e.g. App.xaml.cs / MauiProgram.cs)
ViewModelExtensions.RegisterViewTypes(new Dictionary<string, Type>
{
    ["UserView"]    = typeof(UserView),
    ["OrderView"]   = typeof(OrderView),
    // ... all view types in the application
});

// Navigation continues to call the same method
var viewType = viewModelName.GetRelatedViewType(); // now uses registry
```

### Steps to Migrate

1. In your application's startup code (before first navigation), call `ViewModelExtensions.RegisterViewTypes(IReadOnlyDictionary<string, Type>)` with a dictionary mapping every view type name to its `Type`.
2. If you use source generation (e.g., a custom Roslyn incremental generator), emit the dictionary automatically from all types implementing `IView` in the consuming assembly.
3. Calls to `CancelAllCommands` and `ResetAllCommandStates` that compile under `PublishAot=true` will produce IL2026 warnings — either suppress with a justified `#pragma` or refactor the ViewModel to implement `ICommandProvider` and expose commands explicitly.
4. Calls to `GetEnumDescription(Enum value)` will produce IL2026 warnings; add `#pragma warning disable IL2026` with a justification, or switch to a static lookup dictionary for enum descriptions.
