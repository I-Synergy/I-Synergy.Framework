## AOT Assessment: Compatible

`ISynergy.Framework.Mail` contains 3 source files: `IMailService` (interface), `MailMessage` (model class), and `BaseMailOptions` (options POCO). All are pure contracts and data models with no reflection, no `dynamic`, no `JsonSerializer`, no `Activator.CreateInstance`, and no assembly scanning. The library is fully AOT-compatible as written.

## Migration Guide

`ISynergy.Framework.Mail` is fully AOT-compatible. No code changes or migration steps are required.

### Before

```csharp
// No changes needed
services.AddScoped<IMailService, MyMailService>();
var message = new MailMessage { To = "user@example.com", Subject = "Hello" };
```

### After

```csharp
// Same — IMailService and MailMessage are pure contracts
services.AddScoped<IMailService, MyMailService>();
var message = new MailMessage { To = "user@example.com", Subject = "Hello" };
```

### Steps to Migrate

1. No migration steps required for this library.
2. Verify that the concrete `IMailService` implementation (e.g., `ISynergy.Framework.Mail.Microsoft365` or `ISynergy.Framework.Mail.SendGrid`) is AOT-compatible — see the individual transport migration guides.
3. If you serialize `MailMessage` to JSON, register it with `[JsonSerializable]` in your application's `JsonSerializerContext`.
