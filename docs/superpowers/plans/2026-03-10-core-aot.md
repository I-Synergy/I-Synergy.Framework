# ISynergy.Framework.Core AOT Compatibility Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `ISynergy.Framework.Core` fully AOT-compatible by annotating all reflection-heavy APIs with trimmer/AOT attributes, replacing the JSON-based deep clone with a `JsonTypeInfo`-aware overload, and annotating assembly-traversal helpers so AOT-incompatible callers receive a build-time warning.

**Architecture:** No source generator is required for this library. Every AOT issue falls into one of three annotation-only fix categories: (1) `[RequiresUnreferencedCode]` + `[RequiresDynamicCode]` on methods that call reflection APIs over unknown types, (2) `[DynamicallyAccessedMembers]` flow attributes to preserve reflection targets that are statically known at the call site, and (3) replacing untyped `JsonSerializer.Serialize/Deserialize` with an overload that accepts a caller-supplied `JsonSerializerContext` so the generic `Clone<T>` method can be used from AOT-compiled code without a trimmer warning.

**Tech Stack:** .NET 10 / C# 14, `System.Diagnostics.CodeAnalysis` trimmer attributes (`[RequiresUnreferencedCode]`, `[RequiresDynamicCode]`, `[DynamicallyAccessedMembers]`), `System.Text.Json` source-generation context pattern.

## AOT Issues Found

| File | Line(s) | Pattern |

## Implementation Plan

### Step 1 — Annotate `TypeActivator` (Category A + F)

- [ ] Add `[RequiresUnreferencedCode("Type resolution by assembly-qualified name is not AOT-safe.")]` and `[RequiresDynamicCode("Activator.CreateInstance is not AOT-safe for unknown types.")]` to:
  - `CreateInstance(string assemblyQualifiedName)` (line 14)
  - `CreateInstance(Type targetType)` (line 37)
- [ ] The generic `CreateInstance<T>()` (line 29) is safe if `T` is statically known; add `[DynamicallyAccessedMembers(DynamicallyAccessedMemberTypes.PublicParameterlessConstructor)]` constraint to the `T` type parameter.
- [ ] Add `using System.Diagnostics.CodeAnalysis;` if not already present.

### Step 2 — Annotate `TypeExtensions` (Categories A + B)

- [ ] `GetDefaultValue(this Type type)` (line 57): add `[RequiresUnreferencedCode]` + `[RequiresDynamicCode]` — creates an instance of an unknown value type.

- [ ] `GetInterfaces(this Type _self, bool includeInherited)` (lines 17–21): no annotation needed — `GetInterfaces()` is AOT-safe.

### Step 3 — Annotate `ObjectExtensions` (Categories A + E)

  - Add `[RequiresUnreferencedCode]` + `[RequiresDynamicCode]` to the method signature.
  - `To<T>` (line 50) delegates to `To(value, typeof(T))` — annotate with the same attributes propagated forward.
- [ ] `HasMethod(this object? obj, string methodName)` (line 100): add `[RequiresUnreferencedCode]` to the method.
- [ ] `HasProperty(this object? obj, string propertyName)` (line 122): add `[RequiresUnreferencedCode]` to the method.
- [ ] `FindConversionMethod` (line 204, private): add `[RequiresUnreferencedCode]` to the method.

  - `HasIdentityProperty<T>(this T _self)` — add to `T`
  - `GetParentIdentityPropertyName<T>()` — add to `T`
  - `GetParentIdentityPropertyType<T>()` — add to `T`
  - `GetPropertyValue<T, TResult>()` — add to `T`
  - `GetTitleValue<T>()` — add to `T`
  - `HasParentIdentityProperty<T>()` — add to `T`

## Migration Guide

Reflection-heavy APIs in `ISynergy.Framework.Core` are now annotated with `[RequiresUnreferencedCode]`, `[RequiresDynamicCode]`, or `[DynamicallyAccessedMembers]`. A new AOT-safe `Clone<T>(JsonSerializerContext)` overload is added, and `WeakEventSource<T>` replaces `Expression.Lambda.Compile()` with an open delegate. Callers targeting `PublishAot=true` will see build-time IL2026/IL3050 warnings at call sites that use the annotated APIs.

### Before

```csharp
// Deep clone via reflection-based JsonSerializer
var copy = original.Clone<MyEntity>();

// Resolving a type by name
var instance = TypeActivator.CreateInstance("MyApp.MyType, MyApp");

// Calling a DI service by runtime Type
var service = serviceLocator.GetService(typeof(IMyService));
```

### After

```csharp
// AOT-safe clone using a source-generated JsonSerializerContext
[JsonSerializable(typeof(MyEntity))]
internal partial class AppJsonContext : JsonSerializerContext { }

var copy = original.Clone(AppJsonContext.Default);

// TypeActivator.CreateInstance(string) is now annotated [RequiresUnreferencedCode]
// Prefer the generic overload where T is statically known:
var instance = TypeActivator.CreateInstance<MyType>();

// Prefer generic DI overloads (no annotation needed):
var service = serviceLocator.GetService<IMyService>();
```

### Steps to Migrate

1. Replace `obj.Clone<T>()` calls that run in AOT-published code with the new `Clone<T>(JsonSerializerContext)` overload, passing a `[JsonSerializable]`-registered context for `T`.
2. Replace `TypeActivator.CreateInstance(string)` and `TypeActivator.CreateInstance(Type)` calls with the generic `CreateInstance<T>()` overload wherever `T` is statically known.
3. Replace `GetService(typeof(T))` / `GetRequiredService(typeof(T))` non-generic calls with `GetService<T>()` / `GetRequiredService<T>()`.
4. For any remaining callers of annotated methods (e.g. `CompareUtility.CompareObject`, `DataReaderExtensions.MapToList<T>`), add `#pragma warning disable IL2026` with a justification comment if the call site is in a non-AOT context, or refactor to avoid runtime reflection.
5. If `GetAllReferencedAssemblies` / `GetAllReferencedAssemblyNames` are used, replace them with a statically-built list of assemblies passed at startup.

## No Source Generator Required

All issues in this library are amenable to annotation-only fixes (`[RequiresUnreferencedCode]`, `[RequiresDynamicCode]`, `[DynamicallyAccessedMembers]`) or targeted code changes (replacing `Expression.Compile()` with an open delegate; adding a `JsonSerializerContext` overload). A source generator is not needed because there is no compile-time type discovery or `MakeGenericType` over consumer-defined types.
