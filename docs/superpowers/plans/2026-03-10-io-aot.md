# ISynergy.Framework.IO AOT Compatibility Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix one `JsonSerializer` AOT issue and one `Assembly.GetManifestResourceNames()` reflection call in `ISynergy.Framework.IO`.
**Architecture:** Two files require annotation-only changes. `BaseFileTypeAnalyzer.cs` uses `JsonSerializer.Deserialize<IEnumerable<FileTypeInfo>>(...)` without a `JsonSerializerContext` — this must be replaced with a source-generated `JsonSerializerContext`. `FileTypeAnalyzer.cs` calls `Assembly.GetManifestResourceNames()` which is safe (it doesn't scan arbitrary types) but warrants a comment. No source generator project is needed; a `JsonSerializerContext` partial class is sufficient.
**Tech Stack:** .NET 10, C# 14, `System.Text.Json` source generation.

## Implementation Plan

### Step 1 — Add `JsonSerializerContext` for `FileTypeInfo`

- [ ] Create `src/ISynergy.Framework.IO/Serialization/IoJsonContext.cs`:
  ```csharp
  using System.Text.Json.Serialization;
  using ISynergy.Framework.IO.Models;

  namespace ISynergy.Framework.IO.Serialization;

  [JsonSerializable(typeof(IEnumerable<FileTypeInfo>))]
  [JsonSerializable(typeof(List<FileTypeInfo>))]
  [JsonSourceGenerationOptions(PropertyNameCaseInsensitive = true)]
  internal partial class IoJsonContext : JsonSerializerContext { }
  ```

### Step 2 — Update `BaseFileTypeAnalyzer.LoadFileTypes` to use the context

- [ ] Open `src/ISynergy.Framework.IO/Analyzers/Base/BaseFileTypeAnalyzer.cs`
- [ ] Replace the `LoadFileTypes` method body:
  ```csharp
  // Before:
  if (JsonSerializer.Deserialize<IEnumerable<FileTypeInfo>>(flatFileData, new JsonSerializerOptions
  {
      PropertyNameCaseInsensitive = true
  }) is IEnumerable<FileTypeInfo> fileTypes)

  // After:
  if (JsonSerializer.Deserialize(flatFileData, IoJsonContext.Default.IEnumerableFileTypeInfo)
  ```

### Notes
- A separate source generator project is **not** needed — the `[JsonSerializable]` attribute approach is the standard .NET 10 pattern for AOT-safe JSON deserialization.
- `FileTypeInfo` has only primitive-typed properties (`string`, `int`, `byte[]`) computed from string fields — source generation will work without any additional attributes on `FileTypeInfo` itself.

## Migration Guide

`ISynergy.Framework.IO` replaces the reflection-based `JsonSerializer.Deserialize<IEnumerable<FileTypeInfo>>(...)` call with a source-generated `IoJsonContext`. This is an internal change; no consumer-facing API changes occur.

### Before

```csharp
// BaseFileTypeAnalyzer used reflection-based JsonSerializer.Deserialize
// Caused IL2026 warning when publishing with <PublishAot>true</PublishAot>
var analyzer = new FileTypeAnalyzer();
var fileType = analyzer.Analyze(stream); // internally uses AOT-unsafe JSON
```

### After

```csharp
// Same public API — internal deserialization now uses IoJsonContext (source-generated)
var analyzer = new FileTypeAnalyzer();
var fileType = analyzer.Analyze(stream); // internally uses AOT-safe IoJsonContext
```

### Steps to Migrate

1. No consumer-facing API changes — the `FileTypeAnalyzer` and `BaseFileTypeAnalyzer` public interfaces are unchanged.
2. Rebuild your application against the updated library. The internal JSON deserialization now uses `IoJsonContext.Default.IEnumerableFileTypeInfo` and produces no IL2026 warnings.
3. If you have subclassed `BaseFileTypeAnalyzer` and override `LoadFileTypes`, update your override to use the `IoJsonContext` for any additional `JsonSerializer` calls you make.
